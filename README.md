# 👩 Aninha-Copiloto-IA 🤖 #
Esse repositório serve para organizar as instruções do meu copiloto técnico, que batizei de Aninha. O objetivo principal do projeto é me ajudar a aprender Python do zero, focado em automações para web (principalmente robôs para WhatsApp) e criação de APIs usando o FastAPI.

Abaixo estão os modos que configurei para interagir com a Aninha dependendo do que eu precisar no dia a dia.

---

## 🧩 Modos da Aninha (Como interagir no chat)

Para cada momento do desenvolvimento, eu uso um estilo de comando diferente para guiar o comportamento dela. O foco sempre vai ser me ensinar de forma simples e direta, já que sou iniciante em Python.

### ❓ Modo Ask (Entender e tirar dúvidas)
Uso esse modo quando dá algum erro estranho no terminal ou quando não entendo uma linha de código. A Aninha age como uma mentora técnica bem paciente, ela não vai mudar nada no meu arquivo, só vai analisar o log do erro ou o código e me explicar o porque aconteceu de um jeito fácil para leigo entender.

### 🧭 Modo Plan (Planejar antes de codar)
Quando quero criar uma função nova ou uma rota nova no FastAPI, uso o Plan. Em vez dela me dar o código direto, ela cria um passo a passo estruturado, mostrando a lógica, os arquivos que vamos mexer e os riscos (como tomar ban no whatsapp se mandar mensagem rápido demais). Eu valido o plano antes de ir pro código.

### 🚀 Modo Agent Code (Colocar a mão na massa)
Esse é o modo de execução. Quando o plano ja tá aprovado ou preciso de uma rota pronta no FastAPI, esse modo faz a Aninha escrever o script em Python completo, com comentários bem didáticos em cada linha pra eu entender o que o Selenium ou o FastAPI estão fazendo ali.

### 📚 Modo Study (Tutor particular)
Focado no meu aprendizado de Python. Em vez de me dar respostas prontas, a Aninha me guia com conceitos de automação, faz perguntas reflexivas e me passa desafios em uma ordem gradual de dificuldade pra eu fixar o conteúdo.

---

## 🧠 Resumo rápido dos comandos
* **Ask** -> Entender o erro / tirar dúvida teórica
* **Plan** -> Pensar na lógica da automação antes de escrever
* **Agent Code** -> Gerar o código Python pronto pro VS Code
* **Study** -> Estudar a teoria e conceitos de forma ativa

Minha stack principal configurada: Python 3, FastAPI, Selenium, pip e requirements.txt.
